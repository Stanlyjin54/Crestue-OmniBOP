# 政府采购与社交媒体API测试项目总结

## 项目概述

本项目实现了一个完整的API测试系统，用于测试政府采购信息采集系统和社交媒体商机分析系统的API接口。系统包含模拟后端服务器、前端测试页面和API测试客户端工具。

## 系统架构

### 1. 模拟后端服务器 (mock-server.js)
- **端口**: 6789
- **功能**: 提供政府采购和社交媒体API的模拟数据
- **CORS支持**: 允许跨域请求
- **API端点**:
  - `/api/test/connection` - 连接测试
  - `/api/procurement/list` - 采购项目列表
  - `/api/procurement/crawler/statistics` - 采集统计
  - `/api/social-media/douyin/analyze` - 内容分析
  - `/api/social-media/compliance/check` - 合规检查

### 2. 前端测试页面 (api-test.vue)
- **位置**: `pagesA/api-test/index.vue`
- **功能**: 提供可视化的API测试界面
- **特性**:
  - 连接状态检测
  - 政府采购API测试
  - 社交媒体API测试
  - 完整测试流程
  - 测试结果展示

### 3. API测试客户端 (api-test.js)
- **位置**: `utils/api-test.js`
- **功能**: 封装API请求逻辑
- **方法**:
  - `testConnection()` - 测试连接
  - `request()` - 通用请求方法
  - `runMockTest()` - 运行模拟测试
  - `runRealTest()` - 运行真实测试

## 功能特性

### 政府采购API测试
1. **采购列表测试**
   - 支持分页参数
   - 关键词搜索
   - 返回模拟采购项目数据

2. **采集统计测试**
   - 总采集数量统计
   - 今日采集统计
   - 商机数量统计
   - 成功率统计

### 社交媒体API测试
1. **内容分析测试**
   - 抖音内容分析
   - 产品提及识别
   - 情感分析
   - 参与度评分

2. **合规检查测试**
   - 内容合规性检查
   - 风险等级评估
   - 合规评分

### 测试模式
1. **模拟模式**
   - 使用mock-server.js提供的模拟数据

   - 快速验证功能

2. **真实模式**
   - 连接真实后端服务
   - 使用实际API数据
   - 生产环境验证

## 使用指南

### 快速开始
1. 启动模拟服务器：`node mock-server.js`
2. 启动前端服务：`npx uni dev`
3. 访问测试页面：http://localhost:5173/pagesA/api-test

### 测试步骤
1. **连接测试**: 点击"测试连接"按钮验证后端服务
2. **政府采购测试**: 测试采购列表和采集统计API
3. **社交媒体测试**: 测试内容分析和合规检查API
4. **完整测试**: 运行所有API测试

### 高级功能
- 自定义测试数据
- 添加新API端点
- 扩展测试客户端
- 性能监控

## 技术栈

### 后端技术
- **Node.js**: 服务器运行环境

- **API Test Client**: 自定义API测试客户端

- **Validation Script**: 验证脚本
